Prompt 1 — Системийн үндсэн архитектур ба өгөгдлийн загвар
You are an expert full-stack engineer. Build a production-grade multi-entity marketplace platform called **eseller.mn** using: - Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS - Backend: Next.js API Routes or tRPC - Database: PostgreSQL with Prisma ORM - Auth: NextAuth.js with role-based access control - Storage: Cloudinary (images/media) - State: Zustand + React Query --- ## CORE DATA MODELS ### 1. User & Role System ```prisma model User { id String @id @default(cuid()) email String @unique name String? phone String? role UserRole @default(USER) entityType EntityType? createdAt DateTime @default(now()) store Store? agent Agent? company Company? autoDealer AutoDealer? serviceProvider ServiceProvider? listings Listing[] // user classifieds orders Order[] chats Chat[] } enum UserRole { USER // зар байршуулах эрхтэй хэрэглэгч STORE // дэлгүүрийн эрх AGENT // үл хөдлөхийн агент COMPANY // барилгын компани AUTO_DEALER // авто худалдаа SERVICE // үйлчилгээ ADMIN } enum EntityType { STORE | AGENT | COMPANY | AUTO_DEALER | SERVICE } ``` ### 2. Unified Product/Listing Feed Model ```prisma model FeedItem { id String @id @default(cuid()) refId String @unique // PRD-ELC-010001 format title String description String? price Float? images String[] category Category @relation(...) entityType EntityType // whose item is this entityId String // store/agent/company/autoDealer id tier ItemTier @default(NORMAL) status ItemStatus @default(ACTIVE) viewCount Int @default(0) createdAt DateTime @default(now()) expiresAt DateTime? } enum ItemTier { VIP // ВИП — хамгийн дээр гарах FEATURED // Онцлох DISCOUNTED // Хямдрал NORMAL // Энгийн } ``` ### 3. Entity Profiles (Store / Agent / Company / Auto / Service) All entity profiles share this base structure, extended per type: ```prisma model Store { id String @id @default(cuid()) userId String @unique name String slug String @unique logo String? coverImage String? description String? address String? lat Float? lng Float? phone String? isVerified Boolean @default(false) products Product[] } model Agent { id String @id @default(cuid()) userId String @unique name String slug String @unique profilePhoto String? // pro зураг coverImage String? bio String? licenseNumber String? experience Int? // жилийн туршлага specialties String[] // мэргэшсэн чиглэл phone String? isVerified Boolean @default(false) listings RealEstateListing[] } // Company / AutoDealer / ServiceProvider follow same pattern // with their own extended fields (projects, specs, portfolio) ``` --- ## REGISTRATION FLOW When a user signs up and selects entity type: 1. Choose: Хэрэглэгч / Дэлгүүр / Агент / Барилгын компани / Авто / Үйлчилгээ 2. Fill entity-specific profile form (slug, cover, logo/photo) 3. Admin verifies → `isVerified = true` → full access unlocked 4. Unverified entities can create drafts but not publish Implement `/register` with multi-step wizard: - Step 1: Email/phone + password - Step 2: Select role/entity type - Step 3: Entity profile setup (dynamic per type) - Step 4: Upload cover image + logo/profile photo --- Build this schema with all migrations, seed data for each entity type, and a unified `/api/feed` endpoint that returns paginated FeedItems sorted by: VIP → FEATURED → DISCOUNTED → NORMAL, with filtering by entityType, category, price range.
Prompt 2 — Entity профайл хуудас (Агент / Барилга / Авто / Үйлчилгээ)
## ENTITY PROFILE PAGES Build dynamic profile pages for each entity type at `/[entityType]/[slug]`. All share the same layout shell but render entity-specific content sections. --- ### SHARED PROFILE LAYOUT STRUCTURE ``` /[entityType]/[slug] ├── CoverImage (full-width hero, 1200x400) ├── ProfileCard (overlapping cover bottom) │ ├── Avatar / Logo / Pro photo │ ├── Name + verified badge │ ├── Entity type badge (Агент | Компани | Авто | ...) │ ├── Rating + review count │ ├── Phone reveal button ("Дугаар харах") │ └── Chat / Contact CTA ├── TabNavigation (dynamic per entityType) └── ContentSections (below tabs) ``` --- ### AGENT PROFILE — `/agent/[slug]` Tabs: [ Зарууд | Тухай | Үнэлгээ ] **"Тухай" section includes:** - Bio / мэргэжлийн танилцуулга (rich text) - Туршлага (жилийн тоо) - Мэргэшсэн чиглэл (badges: Орон сууц | Оффис | Газар | ...) - Лицензийн дугаар (verified icon) - Нийт борлуулсан зар тоо (stats) **"Зарууд" section:** - Grid of RealEstateListingCard components - Filter: Зарна | Түрээслүүлнэ | Бүгд - Sort: Шинэ | Үнэ өсөх | Үнэ буурах **Agent can edit their own profile:** - Cover image upload (Cloudinary, 16:9 crop) - Profile photo upload (1:1 crop, circular display) - All bio fields inline edit or modal form --- ### COMPANY PROFILE — `/company/[slug]` (Same for AUTO_DEALER and SERVICE) Tabs: [ Төслүүд | Галлерей | Тухай компани | Баримт бичиг ] **"Тухай компани" section:** - Official description (rich text, editable) - Founded year, employee count, license - Awards / certifications (add/remove list) - Social links (Facebook, Website, etc.) **"Төслүүд" section:** - ProjectCard grid: thumbnail, name, location, status badge (Хэрэгжиж байна | Дууссан | Захиалга авч байна) - Each project links to `/company/[slug]/project/[projectId]` **Project detail page includes:** - Hero gallery (swiper, full-width) - Description, location map - Floor plans (image upload) - Pricing table (unit types + m² + price) - Progress updates (timeline) - Contact / inquiry form **"Галлерей" section:** - Masonry grid, Cloudinary-hosted - Admin of entity can add/remove images - Lightbox on click **"Баримт бичиг" section:** - Uploadable documents (PDF): Зөвшөөрөл, Стандарт, Гэрчилгээ - Each doc: title + upload date + download button - Add/remove by entity owner --- ### OWNER EDIT MODE When `session.user.entityId === profile.id`: - Show floating "Профайл засах" button (bottom-right) - Inline edit toggle: all text fields become editable - Image sections show upload overlay on hover - Section blocks show "+ Нэмэх" / "× Хасах" controls - Save triggers optimistic update + API patch --- Build all profile page components with: - Skeleton loading states - Empty state illustrations per section - Mobile-responsive layout (single column on mobile) - `generateMetadata()` for SEO per profile
Prompt 3 — Нүүр хуудас: нэгдсэн feed, ангилал, tier шүүлт
## HOME PAGE — UNIFIED MULTI-ENTITY FEED Build the main home page `/` that aggregates all entity types into one beautiful, filterable, tiered feed. --- ### PAGE STRUCTURE ``` — rotating VIP/featured promotions — horizontal scroll: Дэлгүүр|Агент|Барилга|Авто|Үйлчилгээ|Зар — хямдрал, шинэ ирэлт, купон мэдэгдэл — "ВИП бүтээгдэхүүн" titled row (paid placement) — "Онцлох" grid (paid featured) — "Хямдрал" horizontal scroll — агент | барилга | авто булангийн онцлох профайлууд — remaining NORMAL items paginated
``` --- ### CATEGORY NAV BEHAVIOR Each category tab filters the feed to show items from that entity type only. URL: `/?category=agent` or `/?category=company` etc. When `category=agent`: - EntitySpotlight shows featured agents (profile card with photo + stats) - MainFeed shows RealEstateListingCards - VIP/Featured sections also filter to agent listings **Entity card variants per type:** ```tsx // Agent card proPhoto, name, verifiedBadge, specialty tags, listingCount, "Зар харах" CTA // Company / Auto / Service card logo, coverThumb, companyName, activeProjectCount / carCount / serviceCount, verifiedBadge, "Профайл харах" CTA // Classifieds (user listing) card image, title, price, location, postedAgo, viewCount, save button ``` --- ### TIER DISPLAY RULES ``` VIP items: - Larger card (2x width in grid) - Gold border accent - "ВИП" badge top-left - Always appear first in their category FEATURED items: - Standard card + blue "Онцлох" badge - Appear after VIP section DISCOUNTED items: - Strike-through original price - Red discount percentage badge - Dedicated horizontal scroll row NORMAL items: - Standard grid, sorted by createdAt desc ``` --- ### FEED API ```typescript // GET /api/feed // Query params: { category?: EntityType, tier?: ItemTier, page?: number, // default 1 limit?: number, // default 20 sort?: 'newest' | 'price_asc' | 'price_desc', search?: string, district?: string, priceMin?: number, priceMax?: number, } // Response: { vip: FeedItem[], // always included regardless of page featured: FeedItem[], discounted: FeedItem[], normal: FeedItem[], meta: { total, page, hasMore } } ``` --- ### PERFORMANCE REQUIREMENTS - VIP/Featured sections: server-side rendered (SSR) - Main feed: client-side infinite scroll with React Query - Images: Next.js Image component with blur placeholder - Category switch: optimistic UI, no full page reload - Implement `loading.tsx` skeleton for each section

Role guard middleware: redirect non-admins to /403 - Audit log: every admin action saved to AuditLog table - Optimistic updates with rollback on error - Export to CSV for all tables

## MONETIZATION SYSTEM Build a complete monetization infrastructure for paid placements, subscription plans, and future extensibility. --- ### PRICING TIERS (configurable from admin) ```typescript // Admin-configurable pricing plans interface PricingPlan { id: string name: string // "ВИП 7 хоног" | "Онцлох 30 хоног" | ... tier: ItemTier // VIP | FEATURED | DISCOUNTED entityTypes: EntityType[] // who can purchase durationDays: number price: number // MNT maxItems: number // хэдэн бараанд зэрэг хэрэглэж болох isActive: boolean features: string[] // display features list } ``` **Default plans to seed:** ``` ВИП байршил: - 7 хоног: 49,900₮ (1 item) - 30 хоног: 149,900₮ (1 item) - 3 сар: 349,900₮ (3 items) Онцлох байршил: - 7 хоног: 19,900₮ (3 items) - 30 хоног: 59,900₮ (5 items) Агент Pro профайл: - 1 сар: 29,900₮ (verified badge + top of search) - 3 сар: 79,900₮ Барилгын компани Pro: - 1 сар: 89,900₮ (unlimited projects + gallery) ``` --- ### PURCHASE FLOW ``` Entity owner → /dashboard/promote → Select item(s) → Choose plan → Payment (QPay / Card) → Immediate activation → Confirmation email + dashboard notification ``` **Payment integration:** - QPay API (primary, Mongolian) - Stripe (card backup) - Invoice generation (PDF) - Webhook: on payment success → activate tier, set expiresAt --- ### ENTITY OWNER DASHBOARD — /dashboard/* ``` /dashboard ├── /overview — stats: views, inquiries, active listings ├── /listings — manage items (agent/zar) ├── /products — manage products (store) ├── /projects — manage projects (company/auto) ├── /promote — buy VIP/Featured placement │ ├── ActivePromotions (expiry countdown) │ ├── PlanSelector (cards with features) │ └── PaymentCheckout ├── /analytics — own item performance ├── /messages — chat inbox ├── /settings — profile, cover, photos └── /billing — invoices, payment history ``` --- ### ADMIN MONETIZATION CONTROLS — /admin/monetization ```tsx — MRR, total revenue, by plan breakdown — Active VIP count, Featured count — Renewal rate, churn — CRUD for all pricing plans — Enable/disable plans per entity type — Promotional pricing (limited time discount) — Bulk plan assignment (admin can manually gift VIP) — Table: entity → plan → start → expiry → revenue — Manual extend / cancel / refund — Export to Excel — 30/60/90 day forecast based on renewals — Scenario planning (if we add X plan...) ``` --- ### NOTIFICATION SYSTEM Trigger notifications for: - VIP/Featured expiring in 3 days → renewal prompt - Payment successful → activation confirmation - New inquiry/chat on promoted item → priority alert - Admin verification approved → welcome + onboarding Channels: in-app bell, email (Resend), SMS (optional) --- ### FUTURE EXTENSIBILITY HOOKS ```typescript // All monetization decisions go through this service // Makes it easy to add new tiers/rules later class MonetizationService { async canPromote(entityId, tier): Promise async activatePromotion(itemId, planId, paymentId): Promise async getActivePromotions(entityId): Promise async scheduleExpiry(promotionId, expiresAt): Promise async handleExpiry(promotionId): Promise // downgrade tier } ``` Build this with full test coverage on MonetizationService, end-to-end purchase flow tests, and admin override capabilities. All pricing must be manageable from admin UI without code changes.
